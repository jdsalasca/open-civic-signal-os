# Execution Checklists

## Implementation

- [ ] Story ID selected.
- [ ] Scope constrained to one objective.
- [ ] Backend/contract/frontend impacts identified.
- [ ] Tests and docs included in same PR.

## Validation

- [ ] Run backend tests.
- [ ] Run frontend tests/build.
- [ ] Run reproducibility checks.
- [ ] Validate OpenAPI/examples updated.

## PR

- [ ] Include civic impact statement.
- [ ] Include risk and rollback plan.
- [ ] Include commands executed.
- [ ] Link related issue/story.

## Release

- [ ] Changelog updated.
- [ ] Known limitations documented.
- [ ] Dashboard explainability fields visible.
- [ ] Audit metadata present in published outputs.
